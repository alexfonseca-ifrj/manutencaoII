Capítulo 1

A árvore de diretórios do Linux

No Linux, tudo parte do diretório raiz, representado por /. A partir dele, o sistema organiza arquivos de usuários, programas, dispositivos, configurações e dados temporários.

/ ├── bin comandos essenciais ├── boot arquivos de inicialização ├── dev dispositivos do sistema ├── etc arquivos de configuração ├── home pastas dos usuários ├── media mídias removíveis ├── mnt montagem manual ├── opt programas opcionais ├── root pasta do administrador ├── tmp arquivos temporários ├── usr programas e bibliotecas └── var logs e arquivos variáveis

Momento-chave

Shell Script transforma comandos repetitivos em automação

Um Shell Script é um arquivo de texto com comandos executados em sequência pelo interpretador Bash. Ele reduz trabalho manual, evita repetição e cria rotinas padronizadas.

criar pastas mover arquivos gerar relatórios validar condições automatizar rotinas
Capítulo 3

Estrutura básica de um Shell Script

O script começa com o shebang, que indica o interpretador usado. Em seguida, podem vir comentários, variáveis, comandos, condicionais e laços.

#!/bin/bash

# Exemplo inicial de Shell Script
echo "Olá, turma!"

DIRETORIO="$HOME/aula_linux"

mkdir -p "$DIRETORIO/scripts"
echo "Pasta criada em: $DIRETORIO/scripts"
#!/bin/bash

echo

Atividade principal

Automação: organizar arquivos por tipo

Situação-problema: um usuário possui vários arquivos misturados em uma pasta chamada downloads_aula. A tarefa é criar um script que organize os arquivos em pastas: documentos, imagens, scripts, planilhas e outros.

1. Preparação da pasta de teste

cd ~
mkdir -p downloads_aula
cd downloads_aula

touch trabalho.txt
touch relatorio.pdf
touch foto.jpg
touch imagem.png
touch rotina.sh
touch dados.csv
touch arquivo_sem_tipo

ls

2. Criar o arquivo do script

cd ~
nano organizar_downloads.sh

3. Código completo do script

#!/bin/bash

# Script para organizar arquivos por tipo
# Aula: Estrutura de Diretórios e Shell Script

PASTA_ORIGEM="$HOME/downloads_aula"
RELATORIO="$PASTA_ORIGEM/relatorio_organizacao.txt"

echo "Iniciando organização da pasta: $PASTA_ORIGEM"

# Verifica se a pasta existe
if [ ! -d "$PASTA_ORIGEM" ]; then
    echo "Erro: a pasta downloads_aula não existe."
    exit 1
fi

# Criação das pastas de destino
mkdir -p "$PASTA_ORIGEM/documentos"
mkdir -p "$PASTA_ORIGEM/imagens"
mkdir -p "$PASTA_ORIGEM/scripts"
mkdir -p "$PASTA_ORIGEM/planilhas"
mkdir -p "$PASTA_ORIGEM/outros"

# Organização por extensão
mv "$PASTA_ORIGEM"/*.txt "$PASTA_ORIGEM/documentos/" 2>/dev/null
mv "$PASTA_ORIGEM"/*.pdf "$PASTA_ORIGEM/documentos/" 2>/dev/null
mv "$PASTA_ORIGEM"/*.jpg "$PASTA_ORIGEM/imagens/" 2>/dev/null
mv "$PASTA_ORIGEM"/*.png "$PASTA_ORIGEM/imagens/" 2>/dev/null
mv "$PASTA_ORIGEM"/*.sh "$PASTA_ORIGEM/scripts/" 2>/dev/null
mv "$PASTA_ORIGEM"/*.csv "$PASTA_ORIGEM/planilhas/" 2>/dev/null

# Move arquivos restantes para "outros"
for arquivo in "$PASTA_ORIGEM"/*; do
    if [ -f "$arquivo" ]; then
        if [ "$(basename "$arquivo")" != "relatorio_organizacao.txt" ]; then
            mv "$arquivo" "$PASTA_ORIGEM/outros/"
        fi
    fi
done

# Geração do relatório
echo "Relatório de Organização" > "$RELATORIO"
echo "Data: $(date)" >> "$RELATORIO"
echo "Pasta organizada: $PASTA_ORIGEM" >> "$RELATORIO"
echo "" >> "$RELATORIO"
echo "Documentos: $(find "$PASTA_ORIGEM/documentos" -type f | wc -l)" >> "$RELATORIO"
echo "Imagens: $(find "$PASTA_ORIGEM/imagens" -type f | wc -l)" >> "$RELATORIO"
echo "Scripts: $(find "$PASTA_ORIGEM/scripts" -type f | wc -l)" >> "$RELATORIO"
echo "Planilhas: $(find "$PASTA_ORIGEM/planilhas" -type f | wc -l)" >> "$RELATORIO"
echo "Outros: $(find "$PASTA_ORIGEM/outros" -type f | wc -l)" >> "$RELATORIO"

echo "Organização concluída!"
echo "Relatório gerado em: $RELATORIO"
ls -R "$PASTA_ORIGEM"

4. Executar o script

chmod +x organizar_downloads.sh
./organizar_downloads.sh
Leitura do código

O que o script utiliza?

Variável

PASTA_ORIGEM guarda o caminho da pasta que será organizada.

Condicional

if [ ! -d ... ] verifica se a pasta existe antes de continuar.

Redirecionamento

2>/dev/null oculta erros quando não existe arquivo de uma extensão.

Laço

for arquivo in ... percorre arquivos restantes e move para “outros”.

Estrutura final esperada

downloads_aula/ ├── documentos/ │ ├── trabalho.txt │ └── relatorio.pdf ├── imagens/ │ ├── foto.jpg │ └── imagem.png ├── scripts/ │ └── rotina.sh ├── planilhas/ │ └── dados.csv ├── outros/ │ └── arquivo_sem_tipo └── relatorio_organizacao.txt
