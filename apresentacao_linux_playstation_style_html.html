Módulo 1

História do Unix e do Linux

A origem dos sistemas operacionais modernos começa no Bell Labs. Entender Unix, GNU e Linux é compreender a base tecnológica da internet, dos servidores, do Android e da computação contemporânea.

1969

Nascimento do Unix

Ken Thompson e Dennis Ritchie iniciam o desenvolvimento do Unix no Bell Labs. O objetivo era criar um sistema simples, eficiente e portátil.

1972–1973

A Linguagem C muda tudo

Dennis Ritchie cria a linguagem C e o Unix é reescrito nela. Isso torna o sistema portátil entre diferentes hardwares.

1983

Projeto GNU

Richard Stallman anuncia o Projeto GNU com a missão de construir um sistema operacional completamente livre.

1991

Nasce o Kernel Linux

Linus Torvalds publica o kernel Linux. Combinado aos programas GNU, nasce o sistema GNU/Linux.

Módulo 2

Distribuições Linux

O kernel Linux sozinho não é um sistema completo. As distribuições combinam kernel, programas, interface gráfica e gerenciadores de pacotes.

1993

Slackware

A distribuição ativa mais antiga. Foco em simplicidade, estabilidade e fidelidade à filosofia Unix.

1993

Debian

Projeto comunitário extremamente estável. Base do Ubuntu, Linux Mint e inúmeras outras distribuições.

1994

Red Hat

Distribuição voltada ao ambiente corporativo. Originou Fedora, CentOS e RHEL.

Genealogia das distribuições

Kernel Linux → Slackware • Debian • Red Hat → Ubuntu • Fedora • Mint • RHEL • openSUSE • Kali Linux.

Família Pacote Gerenciador Exemplos
Debian .deb apt / dpkg Ubuntu, Mint, Debian
Red Hat .rpm dnf / yum Fedora, RHEL, CentOS
Slackware .tgz / .txz pkgtool Slackware
Arch .pkg.tar.zst pacman Arch, Manjaro
Módulo 3

Interface Gráfica

No Linux o usuário escolhe a experiência visual. Diferentes ambientes gráficos permitem personalização, desempenho e produtividade.

GNOME

Minimalista e moderno

Interface limpa, elegante e focada em produtividade. Padrão em distribuições como Ubuntu e Fedora.

KDE Plasma

Extremamente personalizável

Interface poderosa com enorme capacidade de customização. Muito popular entre usuários avançados.

XFCE

Leve e eficiente

Excelente desempenho em computadores modestos. Ideal para sistemas antigos ou máquinas virtuais.

Cinnamon

Tradicional e amigável

Interface confortável para usuários vindos do Windows. Muito usada no Linux Mint.

Módulo 4

Estrutura de Arquivos

O Linux utiliza uma estrutura hierárquica unificada iniciada pela raiz “/”. Tudo é tratado como arquivo — inclusive dispositivos de hardware.

Diretório Função Exemplo
/bin Comandos essenciais ls, cp, cat
/etc Arquivos de configuração /etc/passwd
/home Pastas dos usuários /home/aluno
/usr Programas e bibliotecas /usr/bin
/var Logs e arquivos variáveis /var/log
/dev Dispositivos de hardware /dev/sda

Linux

  • Raiz única “/”
  • Diferencia maiúsculas e minúsculas
  • Hardware tratado como arquivo
  • Estrutura padronizada pelo FHS

Windows

  • Unidades C:, D:, E:
  • Não diferencia maiúsculas
  • Hardware por letras/unidades
  • Estrutura centralizada no sistema
Módulo 5

Terminal Linux

O terminal é uma das ferramentas mais poderosas do Linux. Com poucos comandos é possível navegar, criar arquivos, mover dados e administrar o sistema.

pwd
Mostra o diretório atual onde o usuário está navegando.
ls -l
Lista arquivos com detalhes de permissões, tamanho e data.
cd /home
Navega entre diretórios da árvore de arquivos.
mkdir projetos
Cria uma nova pasta chamada “projetos”.
touch nota.txt
Cria um arquivo vazio rapidamente.
sudo apt update
Atualiza os repositórios do sistema em distribuições Debian.
